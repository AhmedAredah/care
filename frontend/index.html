      Settings
      
    
      
    
      
      Read-only view of the running configuration. Writes will land in
      the next phase. API-key-shaped fields are redacted server-side
      and never leave the host. Path strings (model dirs, work dir,
      etc.) are shown as configured so you can see exactly what the
      app will read on the next pipeline run.
    
      
    
      
    
      
    
      
    
      
  
      

  
  
      
    
      Template builder
      
    
      
      Build a template by drawing diagram and narrative regions on a
      sample report. Source PDFs are rendered into the configured
      work_dir only and are never copied to the public
      export directory. Saved templates are validated before any bytes
      hit disk; existing files cannot be overwritten without explicit
      force=true.
    
      

    
      
      
      1. Load source PDF
      
      
      
        
        
        
      
          
          
        
      
      
      
      
      
    
      

    
      
      
      2. Draw regions / mark identifiers
      
      
      
        Identifier words fingerprint the form so the
        pipeline can pick this template (e.g. "Crash Report",
        "MOSERS"). They are different from the
        narrative start/end anchors on the right, which
        bound where the narrative paragraph begins and ends inside its
        page region.
